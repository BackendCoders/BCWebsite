# 🔧 QUICK FIX: Blog Images Not Showing on Live

## TL;DR - Do This NOW

```bash
# SSH into your live server, then run:
ssh user@yourdomain.com
cd /var/www/yourapp
php artisan storage:link
chmod -R 775 storage/app/public
```

That's it! Images will appear.

## Why Images Don't Show on Live

```
❌ Local (works):           ✅ Production (needs symlink):
   No symlink needed         Must have: public/storage → storage/app/public
   Everything works magic    Without symlink: 404 errors
```

The storage symlink connects the web-accessible directory to where Laravel stores files.

---

## Verification Checklist

After running `php artisan storage:link`, verify:

```bash
✓ Symlink exists:        ls -l public/storage
✓ Images exist:          ls storage/app/public/blogs/
✓ Permissions correct:   stat storage/app/public/blogs/
✓ APP_URL updated:       grep APP_URL .env
```

## If It Still Doesn't Work

1. Is symlink created? → `ls -l public/storage` should show `→`
2. Do files exist? → `ls storage/app/public/blogs/` should list files
3. Are permissions correct? → `stat storage/app/public/blogs/`
4. Is APP_URL correct? → `grep APP_URL .env` (no localhost!)
